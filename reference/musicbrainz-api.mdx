description: "Complete reference for every public function in musicbrainz.el: search, lookup, browse, cover art retrieval, MBID validation, and entity predicate helpers."
---

The `musicbrainz.el` package exposes three primary modes of interaction—search, lookup, and browse—mirroring the structure of the upstream MusicBrainz REST API. All functions return parsed JSON as Emacs Lisp alists, which can be passed directly to `musicbrainz-format` or processed with `let-alist`. Interactive variants (marked `###autoload`) can also be invoked with `M-x`.

---

## Search functions

Search requests query a Lucene-backed index and do not require an MBID. Results include a relevance score. The full [Lucene query syntax](https://lucene.apache.org/core/7_7_2/queryparser/org/apache/lucene/queryparser/classic/package-summary.html#package.description) is supported.

### `musicbrainz-search`

The general-purpose search entry point. Accepts any valid search type from `musicbrainz-search-types`.

```emacs-lisp
(musicbrainz-search type query &optional limit offset)
```

<ParamField path="type" type="string" required>
  Entity index to search. Must be one of the values in `musicbrainz-search-types`: `annotation`, `area`, `artist`, `cdstub`, `event`, `instrument`, `label`, `place`, `recording`, `release`, `release-group`, `series`, `tag`, `work`, `url`.
</ParamField>

<ParamField path="query" type="string" required>
  Lucene search query string. Special characters should be escaped; see `musicbrainz-qeury-special-chars` for the list.
</ParamField>

<ParamField path="limit" type="number" default="1">
  Number of results to return. Valid range is 1–100.
</ParamField>

<ParamField path="offset" type="number" default="0">
  Result offset for pagination.
</ParamField>

```emacs-lisp
;; Search for artists named "Autechre"
(musicbrainz-search "artist" "Autechre")

;; Return up to 10 releases matching "tri repetae"
(musicbrainz-search "release" "tri repetae" 10)

;; Page through results
(musicbrainz-search "label" "Warp" 25 25)
```

---

### `musicbrainz-find`

A general entry point that applies heuristics to decide whether to search or look up. When `query` is a valid MBID it attempts a lookup; otherwise it falls back to a text search.

```emacs-lisp
(musicbrainz-find query &rest extras)
```

<ParamField path="query" type="string" required>
  An MBID or free-text search string.
</ParamField>

<ParamField path="extras" type="string">
  Additional search terms (variadic).
</ParamField>

<Note>
  Heuristic routing is not yet fully implemented. For predictable behaviour prefer `musicbrainz-search` or `musicbrainz-lookup` directly.
</Note>

### `musicbrainz-search-artist`

Search for artists by name and return formatted org-mode table rows.

```emacs-lisp
(musicbrainz-search-artist artist &optional limit)
```

<ParamField path="artist" type="string" required>
  Artist name to search for.
</ParamField>

<ParamField path="limit" type="number">
  Maximum number of results. When omitted only the top result is returned without a score column.
</ParamField>

```emacs-lisp
(musicbrainz-search-artist "Autechre")
;; => ("Autechre | 410c9baf-5469-44f6-9852-826524b80c61 |\n")

(musicbrainz-search-artist "Autechre" 5)
;; => ("100 | Autechre | 410c9baf-5469-44f6-9852-826524b80c61 |\n" ...)
```

### `musicbrainz-artist-to-mbid`

Resolve an artist name to its MBID, returning only 100% confidence matches.

```emacs-lisp
(musicbrainz-artist-to-mbid artist)
```

<ParamField path="artist" type="string" required>
  Artist name to resolve.
</ParamField>

```emacs-lisp
(musicbrainz-artist-to-mbid "Autechre")
;; => "410c9baf-5469-44f6-9852-826524b80c61"
```

<Tip>
  When a name matches multiple artists at 100% (e.g. "Bad Seeds"), use `musicbrainz-disambiguate-artist` to inspect all candidates.
</Tip>

### `musicbrainz-search-label`

Search for record labels, with full disambiguation and lifespan data when `limit` is provided.

```emacs-lisp
(musicbrainz-search-label label &optional limit)
```

<ParamField path="label" type="string" required>
  Label name to search for.
</ParamField>

<ParamField path="limit" type="number">
  Number of results. With a limit, results include score, disambiguation, and date range.
</ParamField>

```emacs-lisp

;; => ("Warp | [[https://musicbrainz.org/label/46f0f4cd...][46f0f4cd...]] |\n")

(musicbrainz-search-label "Music" 5)
;; Returns table with scores, names, descriptions, and date ranges
```

## Lookup functions

Lookup requests require a valid MBID and return detailed data for a specific entity. Subquery variants (`musicbrainz-lookup-ENTITY-SUBQUERY`) include related entities in the response.

### `musicbrainz-lookup`

The general lookup entry point. Validates the entity type and MBID before making the request.

```emacs-lisp
(musicbrainz-lookup entity mbid &optional inc)
```

<ParamField path="entity" type="string" required>
  A core entity type. Must be one of the values in `musicbrainz-entities-core`: `area`, `artist`, `event`, `genre`, `instrument`, `label`, `place`, `recording`, `release`, `release-group`, `series`, `work`, `url`.
</ParamField>

<ParamField path="mbid" type="string" required>
  A 36-character MusicBrainz Identifier (UUID format). Validated by `musicbrainz-mbid-p`.
</ParamField>

<ParamField path="inc" type="string">
  Subquery or relationship include parameter (e.g. `"recordings"`, `"artist-rels"`).
</ParamField>

```emacs-lisp
;; Basic artist lookup
(musicbrainz-lookup "artist" "410c9baf-5469-44f6-9852-826524b80c61")

;; Lookup with subquery
(musicbrainz-lookup "artist" "410c9baf-5469-44f6-9852-826524b80c61" "releases")
```

## Browse functions

Browse requests return all entities directly linked to another entity. Unlike search, they require an MBID and do not rank results by relevance.

### `musicbrainz-browse`

```emacs-lisp
(musicbrainz-browse entity link query &optional type)
```

<ParamField path="entity" type="string" required>
  The result entity type to retrieve (e.g. `"release-group"`).
</ParamField>

<ParamField path="link" type="string" required>
  The linking entity type (e.g. `"artist"`).
</ParamField>

<ParamField path="query" type="string" required>
  The MBID of the linking entity.
</ParamField>

<ParamField path="type" type="string">
  Optional type filter applied to the result entity (e.g. `"album|ep"`).
</ParamField>

```emacs-lisp
;; Browse Autechre's albums and EPs
(musicbrainz-browse "release-group"
                    "artist"
                    "410c9baf-5469-44f6-9852-826524b80c61"
                    "album|ep")

;; Browse all releases on Warp Records
(musicbrainz-browse "release" "label" "46f0f4cd-8aab-4b33-b698-f459faf64190")
```

<Note>
  Browse results are ordered consistently by the linked entity, but not sorted by any field. Fetch all results and sort in Emacs if ordering matters.
</Note>

### `musicbrainz-relations`

A convenience wrapper around `musicbrainz-lookup` for fetching a specific relationship type.

```emacs-lisp
(musicbrainz-relations entity relation mbid)
```

<ParamField path="entity" type="string" required>
  Core entity type to look up.
</ParamField>

<ParamField path="relation" type="string" required>
  Relationship type prefix. The suffix `-rels` is appended automatically. Valid base types include those in `musicbrainz-relationships`: `area`, `artist`, `event`, `instrument`, `label`, `place`, `recording`, `release`, `release-group`, `series`, `url`, `work`.
</ParamField>

<ParamField path="mbid" type="string" required>
  MBID of the entity.
</ParamField>

```emacs-lisp
(musicbrainz-relations "artist" "url" "410c9baf-5469-44f6-9852-826524b80c61")
```

## Cover Art functions

Cover art is served by the [Cover Art Archive](https://coverartarchive.org/) at a separate URL configured by `musicbrainz-coverart-api-url`.

### `musicbrainz-coverart`

```emacs-lisp
(musicbrainz-coverart mbid &optional release-group)
```

<ParamField path="mbid" type="string" required>
  MBID of the release (or release group when `release-group` is non-nil).
</ParamField>

<ParamField path="release-group" type="boolean">
  When non-nil, treat `mbid` as a release-group MBID instead of a release MBID.
</ParamField>

```emacs-lisp
;; Cover art for a release
(musicbrainz-coverart "a929130a-535c-4827-8663-f048e1a7ca0d")

;; Cover art for a release group
(musicbrainz-coverart "fe4acfe9-6d1e-3565-8857-fb16ddc492ab" t)
```

## Utility functions

### `musicbrainz-mbid-p`

Check whether a string is a well-formed MBID (permissive UUID validation).

```emacs-lisp
(musicbrainz-mbid-p mbid)
```

<ParamField path="mbid" type="string" required>
  The string to validate. Must be 36 characters in `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` format using only hexadecimal digits.
</ParamField>

```emacs-lisp
(musicbrainz-mbid-p "410c9baf-5469-44f6-9852-826524b80c61") ;; => t
(musicbrainz-mbid-p "not-an-mbid")                          ;; => nil
```

### `musicbrainz-format`

Pretty-print any API response alist to a string using `pp`.

```emacs-lisp
(musicbrainz-format response)
```

<ParamField path="response" type="list" required>
  An alist returned by any search, lookup, or browse function.
</ParamField>

```emacs-lisp
(musicbrainz-format
 (musicbrainz-browse "release-group" "artist"
                     "410c9baf-5469-44f6-9852-826524b80c61" "album|ep"))
```

### Entity predicate functions

| Function | Description |
|---|---|
| `(musicbrainz-core-entity-p entity)` | Returns `t` if `entity` is in `musicbrainz-entities-core`. |
| `(musicbrainz-non-core-entity-p entity)` | Returns `t` if `entity` is in `musicbrainz-entities-non-core`. |
| `(musicbrainz-uid-entity-p entity)` | Returns `t` if `entity` is a UID type (`discid`, `isrc`, `iswc`). |
| `(musicbrainz-search-type-p type)` | Returns `t` if `type` is a valid search type. |
| `(musicbrainz-linked-entity-p entity)` | Returns `t` if `entity` can be used in a browse request. |
