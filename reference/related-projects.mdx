The `metabrainz` package sits at the intersection of several interconnected open-source projects maintained by the [MetaBrainz Foundation](https://metabrainz.org/). Understanding the broader ecosystem helps clarify what each service provides and how the Emacs package interacts with it. The projects below are either direct dependencies, related databases in the Brainz family, or useful companion tools.

---

## Dependency

### request.el

Both `musicbrainz.el` and `listenbrainz.el` depend on `request.el` for all HTTP communication. It is declared in the package headers:

```emacs-lisp
;; Package-Requires: ((emacs "28.1") (request "0.3"))  ; musicbrainz.el
;; Package-Requires: ((emacs "27.1") (request "0.3"))  ; listenbrainz.el
```

`request.el` provides a synchronous (`:sync t`) HTTP client that blocks Emacs until the response arrives. All functions in this package use the synchronous mode so that return values can be used directly in expressions rather than via callbacks.

Install via MELPA:

```emacs-lisp
(package-install 'request)
```

Or with `use-package`:

```emacs-lisp
(use-package request
  :ensure t)
```

## Query syntax

Search functions in `musicbrainz.el` pass queries directly to MusicBrainz's Lucene-backed search index. The full [Lucene query parser syntax](https://lucene.apache.org/core/7_7_2/queryparser/org/apache/lucene/queryparser/classic/package-summary.html#package.description) is supported, including field-specific searches, boolean operators, wildcards, and fuzzy matching.

```emacs-lisp
;; Field-specific search
(musicbrainz-search "recording" "artist:Autechre AND title:Gantz")

;; Wildcard
(musicbrainz-search "artist" "Aphex*")

;; Fuzzy match
(musicbrainz-search "artist" "Autchere~")
```

<Tip>
  Characters such as `+`, `-`, `&`, `|`, `!`, `(`, `)`, `{`, `}`, `[`, `]`, `^`, `"`, `~`, `*`, `?`, `:`, `\`, and `/` are special in Lucene queries and must be escaped with a backslash when used literally. The full list is in `musicbrainz-qeury-special-chars`.
</Tip>
